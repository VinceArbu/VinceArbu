<h2> Quick Reference Guide for Optimizing Coding Efficiency with Gitüìù</h2>

**1. git init**
<br> -initialize an existing directory as a Git repository.

<br>**Syntax & Parameter:**
<br> git init [none]

<br>**Example:**
<pre>
Assuming you are in the project directory:
1. Open a terminal.
2. Type the command: git init
3. Press Enter.

This will initialize the current directory as a Git repository, allowing you to start tracking changes and versioning your project. </br>
</pre>

<br>**2. git clone [url]**
<br> -retrieve an entire repository from a hosted location via URL.

<br>**Syntax & Parameter:**
<br> git clone [URL]

<br>**Example:**
<pre>
git clone https://github.com/example/repository.git

1. Replace https://github.com/example/repository.git with the actual URL of the Git repository you want to clone.
2. Open a terminal or command prompt.
3. Type the git clone command followed by the repository URL.
4. Press Enter.
  
This will clone the specified Git repository to your local machine, creating a copy of the repository along with its entire version history.
</pre>

<br>**3. git log**
<br> -show the commit history for the currently active branch.

<br>**Syntax & Parameter:**
<br> git log [options] [revision range] [[--] path...]

<br>**Example:**
<pre>
git log                  # Display the entire commit history
git log --oneline        # Condensed one-line view of each commit
git log --author=John    # Show commits by a specific author
git log HEAD~3..HEAD     # Show the last 3 commits
git log --since="2022-01-01"  # Show commits since a specific date
git log path/to/file      # Show commits related to a specific file
</pre>

**4. git add [file]**
<br> -add a file as it looks now to your next commit (stage).

<br>**Syntax & Parameter:**
<br> git add [options] [pathspec]

<br>**Example:**
<pre>
git add myfile.txt          # Add a specific file
git add myfolder/           # Add all changes in a specific folder
git add .                   # Add all changes in the current directory and subdirectories
git add -A                  # Add all changes, including untracked files
git add -u                  # Add all changes except untracked files
</pre>

**5. git reset [file]**
<br> -unstage a file while retaining the changes in the working directory.

<br>**Syntax & Parameter:**
<br> git reset [options] [commit] [--] [file]

<br>**Example:**
<pre>
git reset myfile.txt        # Unstage changes for a specific file
git reset                    # Unstage all changes
git reset --soft HEAD~1      # Reset to the previous commit but keep changes in the working directory
git reset --hard HEAD~1      # Discard changes and reset to the previous commit
</pre>

**6. git stash**
<br> -temporarily save modified and staged changes in the working directory that are not ready to be committed yet.

<br>**Syntax & Parameter:**
<br> git stash [save [-u|--include-untracked] [<message>]]

<br>**Example:**
<pre>
git stash save "Work in progress"     # Stash changes with a message
git stash apply                       # Apply the latest stash
git stash apply stash@{1}             # Apply a specific stash
git stash drop                        # Discard the latest stash
git stash list                        # List all stashes
</pre>

**7. git checkout [branch-name]**
<br> -switch from the current branch to another existing branch.

<br>**Syntax & Parameter:**
<br>git checkout [options] <branch-name> [--] [<pathspec>...]

<br>**Example:**
<pre>
git checkout my-feature-branch       # Switch to an existing branch
git checkout -b new-feature-branch   # Create and switch to a new branch
git checkout -B existing-branch      # Force switch to an existing branch, discarding changes if necessary
git checkout -- myfile.txt            # Restore a specific file in the working directory
</pre>
